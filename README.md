## Overview
This project demonstrates a Web Application Vulnerability Assessment and Penetration Testing (VAPT) performed on an intentionally vulnerable web application for learning and research purposes.

The assessment was conducted using Kali Linux and industry-standard penetration testing tools. The main goal was to identify security vulnerabilities, understand their impact, and provide remediation recommendations based on OWASP Top 10 guidelines.


## Objective
- Identify common web application vulnerabilities
- Perform vulnerability scanning and manual testing
- Validate and exploit security weaknesses
- Analyze risk severity using CVSS scoring
- Map vulnerabilities to OWASP Top 10
- Suggest security improvements


## Methodology
The testing followed a structured penetration testing approach:

1. Reconnaissance and OSINT Enumeration  
2. Application Fingerprinting  
3. Threat Modeling  
4. Vulnerability Scanning  
5. Manual Exploitation  
6. Risk Analysis  
7. Reporting and Documentation  


## Key Vulnerabilities Identified
- SQL Injection (Authentication Bypass and Database Access)
- Blind SQL Injection
- Stored Cross-Site Scripting (XSS)
- Reflected Cross-Site Scripting (XSS)
- Sensitive Information Disclosure
- Local File Inclusion (Path Traversal)
- Brute Force Login Vulnerability
- Weak Password Policy
- Missing Security Headers
- Insecure Cookie Configuration
- Clickjacking
- Insecure HTTP Communication
